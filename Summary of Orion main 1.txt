ORION 3.0 — Personal AI Assistant & Multi-Agent Operator
Author: Abhi | Version: 3.0.0 | Electron + Node.js Desktop App

============================================================
ARCHITECTURE OVERVIEW
============================================================
Stack: Electron 35 + Node.js | Vanilla JS/CSS/HTML | sql.js (SQLite)
AI Backends: Groq (primary), OpenRouter Claude/Grok, Gemini (fallback), Ollama (disabled)

CORE Modules (CORE/):
- main.js: Electron main process, IPC handlers for everything
- ai-brain.js: Multi-brain manager, routes to correct AI backend
- brain-router.js: Smart routing — internet tasks to cloud, code to coder agent, etc.
- memory-engine.js: SQLite session/message/project/folder storage
- personality-engine.js: Mode detection (ORION, FRIDAY, JARVIS, Analyst, Coder, Business)
- agent-registry.js + agent.js + coder-agent.js + business-agent.js + scheduler-agent.js: Sub-agent system
- ollama.js: Local LLM client (disabled)
- user-profile.js: Learns user preferences from interactions
- context-manager.js: Tracks active session context
- argument-engine.js: Pushback and critique engine
- suggestion-engine.js: Proactive AI suggestions
- goal-tracker.js: Goal CRUD with deadlines

UI Files (UI/):
- index2.html: ORION 3.0 3-column Mission Control layout
- renderer.js: Chat, voice, goals, sessions, mode switching
- orion-3.js: View switching, sidebar data binding, progress ring
- styles/orion-3.css: Full dark theme (cyan #00f0ff accent), animations
- styles/orion-theme.css: Legacy v2.0 styles

Key Configs:
- Primary AI: Groq (llama-3.3-70b-versatile)
- Cloud AI: OpenRouter (Claude 3.5 Sonnet, Grok-2), Gemini 2.0 Flash
- Local AI: Ollama DISABLED
- Routing: Cloud-first, Groq primary, Claude/Gemini fallback

============================================================
PHASE 3: BUG FIX — CHAT REPLY NOT WORKING
============================================================
Root Cause: preload.js used variable "initialized" without declaring it.
Missing "let initialized = false;" caused ReferenceError every time
window.orion.getCurrentSessionId() was called.

Impact: sendMessage() crashed, user saw "No chat selected" error,
no AI reply ever appeared.

Fixes:
1. preload.js line 9: Added "let initialized = false;"
2. ai-brain.js: Added non-streaming fallback (_groqNonStreaming)
   for environments where streaming response.body is unavailable

Result: Groq API works, chat replies correctly.

============================================================
AI ROUTING CHAIN
============================================================
User Message
  → Is internet task? → OpenRouter Grok (real-time)
  → Ollama available & localFirst? → Ollama (various models)
  → Default: Fallback chain
      → Groq (primary, llama-3.3-70b-versatile)
      → OpenRouter Claude (claude-3.5-sonnet)
      → OpenRouter Grok (grok-2)
      → Gemini (gemini-2.0-flash)
      → Error: "No AI brain available"

============================================================
MESSAGE FLOW
============================================================
Renderer(sendMessage)
  → preload.js (chat bridge)
    → main.js (IPC: orion:chat)
      → AIBrain.asyncChat()
        → AgentRegistry.executeTask() [check sub-agents]
        → BrainRouter.route() [pick AI backend]
        → _groqStream() [fetch with stream:true]
          → _readOpenAIStream() [parse SSE chunks]
            → onChunk → IPC 'orion:chunk' → renderer
        → Return full response
      → MemoryEngine.saveMessage()
      → ContextManager.update()
      → UserProfile.learn()
    ← Return { response, mode, brain }
  ← renderer updates chat UI
